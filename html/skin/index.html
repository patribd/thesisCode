Remote testbed for measuring the performance of wireless technologies


The increasing use of Wireless Sensor Networks (WSNs) as Internet of Things (IoT) spreads has turned the importance of choosing efficient devices to compose those networks even greater.

For that reason, a way of controlling wireless devices was required to be developed. The main aim of this project is to develop an efficient and secure remote testbed. The wireless communication and private network basically consists of a set of network devices that are monitored by an external device that does not interfere with their behavior, which in this case is the central server of the system.

The main aim of this project is to develop a testbed capable of providing information related to different measurements (e.g. latency, energy, throughput…) that requires no physical interaction from the customer. In order to obtain that result, different Odroids M1S will be in charge of obtaining data that will be stored in the system’s database for future studies. The data received by the central server will be represented in a website so future researches can make their own studies and decisions according to the results obtained. In addition, a second objective was to implement specific configurations in the wireless communication devices, nRF Dongles, configurations specified in a certain binary file. For doing so, a Python script that connects to the wireless devices from the Odroids M1S will be in charge of implementing the new configuration.

The correct functioning of the remote testbed was achieved in the lab, but a series of difficulties were presented while computing the system that prevented the system from being implemented for external networks.
